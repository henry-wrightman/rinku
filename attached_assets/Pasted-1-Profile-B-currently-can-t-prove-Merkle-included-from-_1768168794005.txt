2) Profile C’s validatorProof is still underspecified

You say “MerkleSumTree multi-proof,” but the schema lists:

WHITEPAPER (6)

A multiproof is only verifiable if the verifier knows where each auxiliary node belongs (or you define a deterministic “flags/queue” format). Right now, an implementer can’t write verifyValidatorProof() from this schema without guessing.

Fix options:

Easiest to understand: auxiliaryNodes: Array<{ level: uint8; index: uint16; hash: string; sumWeight: uint64 }>

More compact: define a packed multiproof encoding (“flags + proofNodes”) and reflect that in the schema (“auxiliaryNodesPacked: base64url”).

You already mention canonical JSON and DoS bounds — this is the last missing “unambiguous encoding” piece.
