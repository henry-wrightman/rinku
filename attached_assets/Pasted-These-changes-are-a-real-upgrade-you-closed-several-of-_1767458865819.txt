3) The MerkleSumTree leaf hash uses string concatenation; define canonical byte encoding

This is subtle, but important: in A.3 you define hashes like:

SHA256("leaf:" || index || ":" || address || ":" || base64url(blsPubKey) || ":" || weight)
``` :contentReference[oaicite:11]{index=11}

String concatenation with `:` delimiters is easy to implement, but it’s also where cross-language differences sneak in (number formatting, leading zeros, UTF-8 vs something else, etc.).

**Fix (simple):** define it as byte encoding:
- `index` and `weight` as fixed-width big-endian uint64
- `address` as 20 bytes
- `blsPubKey` as 96 bytes
- prefix tags as 1 byte domain separators (or short ASCII)

You already have “big-endian byte order” rules :contentReference[oaicite:12]{index=12} — apply them here explicitly.
